# 11. 假設與風險

> 回上一層：[README.md](./README.md)

## 11.1 假設

| 編號 | 假設內容 |
|---|---|
| AS-01 | 候選人使用的瀏覽器支援 `getDisplayMedia()` 與 `MediaRecorder`（現代 Chrome/Edge/Firefox） |
| AS-02 | 候選人作答期間網路連線穩定，足以支撐定期自動儲存與 AI 對話請求 |
| AS-03 | Gemini API 免費額度（RPM/RPD）足以支撐 MVP 階段的預期使用量 |
| AS-04 | 主管僅需單一角色權限，暫不需要多層級管理員或協作審閱 |
| AS-05 | 候選人願意配合螢幕錄影同意流程，不會大量拒絕授權 |

## 11.2 風險與因應

| 編號 | 風險 | 影響 | 因應措施 |
|---|---|---|---|
| RI-01 | Gemini 免費額度超額導致 AI 功能暫時不可用 | AI 產題/助手/評分中斷，影響候選人體驗 | 加上 rate limiting 與友善錯誤訊息，必要時提供人工降級（跳過評分） |
| RI-02 | 候選人中途取消螢幕分享導致錄影不完整 | 主管無法完整回放作答過程 | 偵測 `track.onended` 並提示重新授權；錄影缺口於詳情頁明確標示 |
| RI-03 | 候選人使用不支援的瀏覽器 | 無法進行螢幕錄影或編輯器異常 | 進入頁偵測相容性並提示建議瀏覽器 |
| RI-04 | AI 自動評分可能有偏誤或幻覺 | 主管誤判候選人能力 | 明確標示「僅供參考，非最終定論」，UI 上不可弱化此提示 |
| RI-05 | 連結被轉發給非預期第三人 | 面試被冒名代考 | Token 高隨機性 + 效期限制；後續可評估加入基本身分確認機制 |
| RI-06 | 螢幕錄影/程式碼涉及候選人隱私與資料保存 | 法遵/資安疑慮 | Signed URL 限時存取、RLS 隔離、後續需訂定保存期限與刪除政策 |
| RI-07 | 候選人斷網造成自動儲存遺漏 | 作答內容遺失，體驗不佳 | Debounce 儲存 + 前端本地暫存雙保險，並提示儲存狀態 |
