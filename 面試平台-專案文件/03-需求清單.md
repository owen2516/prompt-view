# 3. 需求清單

> 回上一層：[README.md](./README.md) ／ 對應追溯：[12-追溯矩陣.md](./12-追溯矩陣.md)

## 3.1 功能性需求（Functional Requirements）

| 編號 | 需求描述 | 優先度 |
|---|---|---|
| FR-01 | 主管以 Email/Password 登入 | Must |
| FR-02 | 手動建立/編輯/刪除/排序題目 | Must |
| FR-03 | AI 產生結構化題目（標題/描述/起始碼） | Must |
| FR-04 | 建立面試模板（多題組合、時間限制、到期日） | Must |
| FR-05 | 產生一次性 token 連結，並可重新生成 | Must |
| FR-06 | 候選人以連結免登入進入面試 | Must |
| FR-07 | 候選人程式碼自動定期儲存 | Must |
| FR-08 | 候選人中斷後可接續同一 session 繼續作答 | Must |
| FR-09 | 候選人與 AI 助手對話（含完整解答能力） | Must |
| FR-10 | 候選人螢幕錄影並上傳（支援多段） | Must |
| FR-11 | 候選人提交後鎖定作答並轉為完成狀態 | Must |
| FR-12 | 提交後 AI 自動評分（總分+摘要） | Should |
| FR-13 | 提交後 Email 通知主管 | Should |
| FR-14 | 主管檢視面試紀錄列表（含篩選/搜尋/統計） | Must |
| FR-15 | 主管檢視單一面試詳情（程式碼/對話/評分/錄影） | Must |
| FR-16 | 主管標記審閱狀態 | Should |
| FR-17 | 應試者彙整頁面（跨面試模板查詢） | Should |
| FR-18 | AI 設定頁（全域預設模型與 system prompt） | Should |
| FR-19 | 線上程式碼執行/測試 | Could（Phase 5） |
| FR-20 | AI 評分多維度拆分 | Could（Phase 5） |
| FR-21 | 多 AI 模型可選（Claude/GPT 等） | Could（Phase 5） |
| FR-22 | 逾時自動提交、狀態自動轉換 | Could（Phase 6） |
| FR-23 | 面試紀錄匯出（PDF/CSV） | Could（Phase 6） |

## 3.2 非功能性需求（Non-Functional Requirements）

| 編號 | 需求描述 |
|---|---|
| NFR-01 | 候選人端所有寫入須經後端驗證 token 狀態，禁止前端以 anon key 直接寫入 |
| NFR-02 | Supabase RLS 確保主管僅能存取自己建立的資料 |
| NFR-03 | AI API Key 僅存於後端環境變數，不可暴露於前端 |
| NFR-04 | 候選人端 AI 對話與主管端 AI 呼叫需有基本 rate limiting |
| NFR-05 | 錄影檔透過 Signed URL 存取，具時效性 |
| NFR-06 | Email 通知失敗不可阻塞候選人提交流程（非同步、失敗降級） |
| NFR-07 | AI 呼叫超出免費額度時顯示友善訊息，不暴露原始錯誤 |
| NFR-08 | 連結 token 需高隨機性（如 nanoid(24)），不可猜測 |

## 3.3 明確不做（Out of Scope，MVP 階段）

- 問答題／選擇題型（僅支援程式題）
- 候選人登入機制（僅以連結 token 識別）
- 前端本機執行程式碼（一律不在瀏覽器本機執行）
- 多 AI 供應商選擇（MVP 固定 Gemini）
- AI 評分多維度拆分（MVP 僅單一總分）
- 面試官多人協作/多角色權限（MVP 僅單一「主管」角色）
