# 2. User Story + 驗收條件

> 回上一層：[README.md](./README.md)

格式：`US-<角色代碼><編號>`，角色代碼 A=主管（Admin）、C=候選人（Candidate）。

## US-A1：登入後台

**身為** 主管，**我想要** 用 Email/Password 登入後台，**以便** 存取我建立的題目與面試紀錄。

驗收條件：
- [ ] 輸入正確帳密可登入並導向 `/admin/interviews`
- [ ] 輸入錯誤帳密顯示錯誤訊息，不可登入
- [ ] 未登入直接存取 `/admin/*` 會被導向登入頁

## US-A2：AI 產生題目

**身為** 主管，**我想要** 輸入職缺主題與難度後由 AI 產生題目，**以便** 節省手動出題時間。

驗收條件：
- [ ] 輸入主題/難度/程式語言後呼叫 AI，回傳含標題、Markdown 描述、起始程式碼的結構化題目
- [ ] 產生結果可在加入題目列表前編輯或捨棄
- [ ] AI 呼叫失敗（含超過免費額度）時顯示友善錯誤訊息，不可讓使用者看到原始 API 錯誤

## US-A3：產生並管理面試連結

**身為** 主管，**我想要** 為一份面試模板產生專屬連結並設定到期日，**以便** 分享給候選人作答。

驗收條件：
- [ ] 可選擇 7/14/30/60 天有效期限
- [ ] 「重新生成連結」後，舊連結立即失效（狀態變 `expired`），新連結可正常使用
- [ ] 連結卡片顯示開啟次數、完成次數、完成率

## US-A4：審閱候選人結果

**身為** 主管，**我想要** 在詳情頁同時看到最終程式碼、AI 對話紀錄、AI 評分與錄影回放，**以便** 快速且全面地評估候選人。

驗收條件：
- [ ] 三個資訊區塊（程式碼／題目／AI 對話）同版面呈現，不需切換頁面
- [ ] 錄影可播放，若有多段依時間序列出
- [ ] 點擊「標記為已審閱」後，列表與紅點提示數量即時更新

## US-C1：透過連結進入面試（含斷點續答）

**身為** 候選人，**我想要** 用收到的連結開始或接續作答，**以便** 不用擔心中途斷線需要重來。

驗收條件：
- [ ] 連結有效且未完成時，可正常進入
- [ ] 已存在 `in_progress` session 時，載入先前程式碼與 AI 對話，不重置
- [ ] 連結為 `completed`／`expired` 時拒絕進入並顯示對應訊息

## US-C2：撰寫程式碼並自動儲存

**身為** 候選人，**我想要** 我的程式碼被自動儲存，**以便** 不會因瀏覽器當機或斷網而遺失作答。

驗收條件：
- [ ] 編輯後於 debounce 時間內（例：10–30 秒）自動呼叫儲存 API
- [ ] 重新整理頁面後可還原最後一次自動儲存的內容
- [ ] 儲存失敗需有可見的提示（避免候選人誤以為已存檔）

## US-C3：與 AI 助手討論

**身為** 候選人，**我想要** 在作答時詢問 AI 助手，**以便** 釐清題意或取得思路協助。

驗收條件：
- [ ] AI 回覆會參考當前題目與程式碼上下文
- [ ] 每則訊息即時存入對話紀錄，且互動次數即時累加
- [ ] 若候選人要求完整解答，AI 依該面試設定可直接提供（不強制隱藏答案）

## US-C4：提交面試

**身為** 候選人，**我想要** 提交後有明確的完成畫面，**以便** 確定我的作答已成功送出。

驗收條件：
- [ ] 提交前彈出確認 dialog，避免誤觸
- [ ] 提交後停止錄影並上傳，狀態轉為 `completed`
- [ ] 提交後無法再修改或重新進入該連結繼續作答
