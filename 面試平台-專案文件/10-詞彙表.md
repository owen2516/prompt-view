# 10. 詞彙表

> 回上一層：[README.md](./README.md)

| 詞彙 | 定義 |
|---|---|
| 主管（Admin） | 建立題目、面試模板、產生連結、審閱結果的角色，需登入 |
| 候選人（Candidate） | 透過連結進入作答的角色，不需登入，以 token 識別身分 |
| 面試模板（Interview Set） | 一組題目的組合（`interview_sets`），可產生多個連結供不同候選人使用 |
| 面試連結（Interview Link） | 候選人存取面試的一次性網址，內含唯一 `token`，可設定效期 |
| Token | 高隨機性字串，是候選人身分與連結有效性的唯一識別依據 |
| 面試 Session | 候選人單次作答歷程（`interview_sessions`），記錄起訖時間、最終作答、狀態 |
| 斷點續答 | 候選人中斷後重新開啟同一連結，接續原本 in_progress session 而非重新開始 |
| AI 助手 | 候選人作答時可對話的 AI，定位為輔助角色，可視需要提供完整解答 |
| AI 自動評分 | 候選人提交後由 AI 依程式碼與對話產出的總分與摘要，僅供主管參考 |
| 審閱狀態（Review Status） | 主管是否已檢視某筆面試紀錄的標記（pending／reviewed） |
| Signed URL | Supabase Storage 產生的限時有效連結，用於安全播放錄影檔 |
| RLS（Row Level Security） | Postgres 資料列層級權限控管，確保主管僅能存取自己建立的資料 |
