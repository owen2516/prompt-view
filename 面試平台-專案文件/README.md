# 面試平台 專案文件索引

> 本資料夾將 12 個項目各自獨立成檔，方便單獨檢視、維護與分享。技術細節（Schema、API、頁面規格）仍以 [../面試平台-規格開發書.md](../面試平台-規格開發書.md) 為主，本資料夾聚焦「為什麼做、做給誰、怎麼驗證、怎麼追蹤」。

| # | 檔案 | 內容 |
|---|---|---|
| 1 | [01-問題陳述.md](./01-問題陳述.md) | 明確專案的目的和動機 |
| 2 | [02-UserStory與驗收條件.md](./02-UserStory與驗收條件.md) | 定義用戶需求和完成標準 |
| 3 | [03-需求清單.md](./03-需求清單.md) | 詳細列出需要和不需要的功能 |
| 4 | [04-範圍與分期.md](./04-範圍與分期.md) | 規劃專案的階段性目標 |
| 5 | [05-功能清單.md](./05-功能清單.md) | 列出所有需要開發的功能 |
| 6 | [06-流程圖與時序圖.md](./06-流程圖與時序圖.md) | 展示系統或流程的順序和邏輯 |
| 7 | [07-ERD與欄位定義.md](./07-ERD與欄位定義.md) | 資料結構和欄位的設計 |
| 8 | [08-規格書.md](./08-規格書.md) | 詳細的技術和操作規範（索引） |
| 9 | [09-測試案例.md](./09-測試案例.md) | 確保功能符合需求的測試方案 |
| 10 | [10-詞彙表.md](./10-詞彙表.md) | 統一專案中使用的術語，避免誤解 |
| 11 | [11-假設與風險.md](./11-假設與風險.md) | 識別潛在風險和假設 |
| 12 | [12-追溯矩陣.md](./12-追溯矩陣.md) | 追蹤需求與交付成果的對應關係 |

修改任一檔案時，若牽涉資料庫欄位或 Phase 分期，請同步檢查 [../面試平台-規格開發書.md](../面試平台-規格開發書.md) 是否需要一併更新。
