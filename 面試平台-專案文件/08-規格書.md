# 8. 規格書

> 回上一層：[README.md](./README.md)

完整功能規格（頁面版面、欄位、互動細節）已詳列於 [../面試平台-規格開發書.md 第 5–9 節](../面試平台-規格開發書.md)，涵蓋：

- 第 5 節：主管端／候選人端逐頁功能規格
- 第 6 節：AI 整合設計（產題／助手／評分／模型設定）
- 第 7 節：線上程式碼執行與完成通知
- 第 8 節：螢幕錄影實作細節
- 第 9 節：安全性設計

本文件不重複抄錄全文，僅在此列出**跨頁面共通規則**，作為規格書的補充索引：

| 規則 | 說明 |
|---|---|
| 狀態一致性 | `interview_sets.status`、`interview_links.status`、`interview_sessions.status` 三者獨立，UI 呈現時不可混用同一個狀態標籤元件 |
| 寫入權限 | 候選人端所有寫入 API 必須先驗證 `interview_links.status` 為 pending/in_progress，否則一律拒絕 |
| AI 呼叫封裝 | 產題／對話／評分三種用途統一透過 `/lib/gemini` 封裝，禁止在頁面元件內直接呼叫 Gemini SDK |
| 非同步作業 | AI 評分、Email 通知皆為非同步且互不阻塞，任一失敗都不可影響候選人「提交成功」的回應 |
